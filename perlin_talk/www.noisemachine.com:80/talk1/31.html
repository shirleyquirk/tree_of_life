Hardware performance

  • FPGA implementation
  • Fewer than 10000 gates

  • Pipeline performance:
    • one evaluation per instruction
    • 20 instructions latency

  • Result: 200M evals per 10K gates
The implementation is pipelined, so that at every machine instruction you can feed in a new x,y,z triple. About 20 instructions later the noise evaluation for that x,y,z triple pops out. Each input coordinate consists of 8 integer bits plus 8 fractional bits; the result is an 8 bit value. This implementation handles noise over blocks of 2563, creating a repeating tile of noise function. The whole thing fits into under 10000 gates.

For example, on a 200 Mhz chip you could use it to do about 30 million evaluations (or roughly 200/7 million) of seven octave turbulence texture per second. This is on the order of one per pixel in real time, for each 10,000 gates of logic allocated on the accelarator chip.
