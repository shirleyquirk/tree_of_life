back The diagram on the left shows the different phases of my hardware implementation of 3D noise. Each block of the schematic represents an area of the chip.

The architecture is pipelined - at each machine instruction a different noise evaluation is being computed in each circuit of the chip. By clicking on the buttons below, you can hilight in turn the successive parts of the pipeline:

input high order bits i,j,k 
compute 8 gradients L
input low order bits u,v,w
compute 8 grid values H
compute ease curves S
interpolate 
RESET
The tricky part, and the thing that allows me to implement 3D noise with very few gates, is to avoid multiplies in the L blocks and the H blocks. Each of the I blocks requires one multiply, for a total of seven multiplies.

Yes, there are patents pending on all this. :-)
